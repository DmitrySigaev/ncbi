Prohibited C++ and STL features



Prohibited STL features

Don't use hash or the rope classes from some versions of the STL. These are non-standard additions. If you have questions about what is/isn't in the library, consult the draft standard.



Prohibited C++ features

Don't use operator overloading, unless you are writing a very basic class, like string. It turns out that in large projects, people have different ideas of what an overloaded operator means, leading to all sorts of bugs. For example, what exactly does "string1 = string2;" mean?

Never use "using namespace XXX;". Yet, the definition or namespace is allowed. For more details, see in Name prefixing and/or the use of namespaces.
