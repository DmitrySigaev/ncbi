wxWindows setup changes

We made the following changes in the master setup file include\wx\msw\setup.h of wxWindows library:
  • #define wxUSE_SOCKETS 0 to exclude (not to double) the network stuff, which is normally supported in the standard core library, therefore class wxSocket disabled.
  • #define wxUSE_THREADS 0 as the multi-thread support is weak in the wxWindows library.
  • #define wxUSE_GLCANVAS 1 to enable GUI applications to use OpenGL library.
  • #define wxUSE_IOSTREAMH 0 to eliminate conflicts with the rest of the C++ Toolkit, which uses new <iostream> library.
