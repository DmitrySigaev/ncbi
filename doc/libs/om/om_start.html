Object Manager.

Framework setup.

This document describes framework setup for UNIX operating system.
First step in using Object Manager is creation of a separate project folder and a makefile. On CVS C++ tree we have created a special folder, in which we would suggest putting your first Object Manager - related projects. There is also a sample project, which could serve as a starting point.
So, what you should do first is

  • extract Object Manager sample project from the C++ tree,
  • create your own project folder,
  • copy sample project files into your folder,
  • generate a makefile,
  • build the project.

1. Create a working folder. Let us suggest this folder is called omlab and is located in your home directory. 

mkdir ~/omlab
cd ~/omlab

2. Extract Object Manager sample project from the C++ tree. 

$NCBI/c++/scripts/import_project.sh internal/objmgr_lab

3. Create your own project folder - we suggest using your user name, so let us call it userid, - add it to the CVS tree: 

cd ~/omlab/internal/c++/src/internal/objmgr_lab
mkdir userid
cvs add userid

4. Copy sample files, add them to the CVS tree as well. 

cd userid
cp ../sample/Makefile.in .
cp ../sample/Makefile.task.app .
cp ../sample/task.cpp .
cvs add Makefile.in Makefile.task.app task.cpp
cvs commit -m "Object Manager sample project. Initial revision"

5. Generate a makefile using import_project.sh shell script 

cd ~/omlab
$NCBI/c++/scripts/import_project.sh internal/objmgr_lab

6. Build your project. 

cd ~/omlab/internal/c++/src/internal/objmgr_lab/userid
make

7. Run the sample application. 

./omtask1 -gi 2
