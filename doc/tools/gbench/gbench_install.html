Introduction

GBench is a complex multi component product includes several executable, configuration files and plugin modules. It requires a formal installation process which creates gbench directory structure and places files in it.

Windows Installation

On Windows NCBI Genome Workbench includes a special installation project (MSVC). It is located in compilers\msvc_prj\gbench\gbench_install. If you run build this project it will create a full Gbench installation inside of the target directory (DebugDLL or ReleaseDLL) When it is there you can take all the subdirectories and simply copy it at the place of your choice. Genome Workbench remains fully functional when moved from one directory to another.

Unix Installation

NCBI Tollkit includes gbench installation script (gbench_install.sh) for Unix.

Location: c++/scripts/gbench_install.sh
Synopsis: Genome workbench installation. Creates Genome Workbench directory structure and copies all files from the NCBI Toolkit build.

Main arguments

Argument Effect Comments
--copy Copies files instead of making symbolic links. By default installation script creates links on files located in the build directory.
sourcedir path to pre-built NCBI Toolkit  
targetdir Genome Workbench target installation directory  

If you need to move installtion you can recursively copy all the installation directory to the new place and run move-gbench.sh script from the Genome Workbench bin directory. This script will correct NCBI_GBENCH_INSTALL_DIR variable in run-gbench.sh.
