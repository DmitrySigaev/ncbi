Starting new C++ module(source and headers)

Usually, each C++ module consists of 3 files:
  • Header file(*.hpp) -- API for the external users. Ideally, this file contains only declarations of things(with good comments) that are to be used by external users. No less, and no more.
  • Inline file(*.inl) -- All inline function definitions go here, along with the declarations of internally(or indirectly) used things that are of no interest the external user but still must be included as a part of the header file(see example ).
  • Source file(*.cpp) -- Definitions of non-inline functions and internally used things that should not be included to other modules.
Each and every source file must include NCBI disclamer and (preferably) CVS info. Then, the header file must be protected from double-include, and it must include "*.inl" file, if any.

The above examples contain a standard startup frameworks so that one could just cut&paste one of them to start a new C++ file (just dont forget to replace the "framewrk" stubs by your new module name).
