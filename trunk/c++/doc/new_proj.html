Overview of the C++ CVS tree structure

The standard CVS location for the newcoming NCBI C++/STL projects is $CVSROOT/internal/cpp/. It has the following structure:

doc/

"*.html", "readme.*", "*.txt", etc.

Here go the NCBI C++ style and naming conventions, as well as recommendations and directives of what features of C++ can be used(and how) -- for everybody writing on C++ in NCBI. Here may also be a list of referencies to the documentations in the projects/*/doc/ dirs.

projects/makefile.*

Platform-specific makefiles that recursively build libs & stuff in the projects/*/ dirs using the project-related makefiles "projects/*/makefile.*"

projects/*/makefile.*

Platform-specific makefiles to build this particular project.

projects/*/doc/

Documentation for this particular project.

projects/*/include/

Externally used include files("*.hpp" and "*.inl") that describe the API provided by the project(included with #include <*.*>).

projects/*/src/ Source files("*.cpp") that implement the API; and, probably, some internally used include files ("*.hpp" and "*.inl", included with #include "*.*")



Adding CVS directory for a new C++ project

Create your new project directories in the NCBI CVS tree: 

cd $CVSROOT/internal/cpp/projects
mkdir MyNewProject
cd MyNewProject
mkdir include
mkdir src
mkdir doc
Now you can checkout the new directory to some local directory, e.g.: 
cd ~/
cvs checkout -d MyNewProject internal/cpp/projects/MyNewProject
cd MyNewProject
and add and edit your C++ files in there.



Using CVS

For the detailed info on CVS see CVS online manual or use commands "man cvs" or "cvs -H" on your workstation.
