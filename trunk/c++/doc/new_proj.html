CVS tree structure of NCBI C++ projects

 (For the overall NCBI C++ CVS tree structure see in NCBI C++ source code.)

The standard CVS location for NCBI C++/STL projects is "$CVSROOT/internal/cpp/projects/". It contains:

  • "/makefile.*" -- Platform-specific makefiles that recursively build libs & stuff in the "projects/*/" dirs using the project-related makefiles "/*/makefile.*"
  • "/*/makefile.*" -- Platform-specific makefiles to build a particular project.
  • "/*/include/" -- Externally used include files("*.hpp" and "*.inl") that describe the API provided by the project(included with #include <*.*>).
  • "/*/src/" Source files("*.cpp") that implement the API; and, probably, some internally used include files ("*.hpp" and "*.inl", included with #include "*.*")
  • "/*/test/" -- A "test suite"(set of plain applications to test the project's functionality).
  • "/*/doc/" -- Documentation on this project.


Adding CVS directory for a new C++ project

Create your new project directories in the NCBI CVS tree: 

cd $CVSROOT/internal/cpp/projects
mkdir MyNewProject
cd MyNewProject
mkdir include
mkdir src
mkdir doc
Now you can checkout the new directory to some local directory, e.g.: 
cd ~/
cvs checkout -d MyNewProject internal/cpp/projects/MyNewProject
cd MyNewProject
and add and edit your C++ files in there.
