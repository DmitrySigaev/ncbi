Basic integer types

It was decided that everyone must use the standard C/C++ types as much as possible, having in mind the following assumptions of their sizes and limits:

Name Size(bytes) Min Max Note
char 1 ? ? It can be either signed or unsigned!
signed char 1 -128 127
unsigned char 1 0 255
signed short, short 2 or more -32768 or less 32767 or greater Use "int" if size isn't critical
usigned short 2 or more 0 65535 or greater Use "unsigned int" if size isn't critical
int, signed int 4 or more -2147483648 or less 2147483647 or greater
unsigned int 4 or more 0 4294967295 or greater

It is not recommended to use "long" type unless it is absolutely necessary(e.g. in the lower-level code and when third-party code is used); and even if you have to -- try to localize the "long" as much as possible. The use of "long long" type is now prohibited in the portable code.



Auxiliary types

Use the "bool" type to represent boolean values. It accepts one of { false, true }.

Use the "size_t" type to represent a size of memory structure, e.g. obtained as a result of sizeof operation.
